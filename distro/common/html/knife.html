ceth is a command-line tool that provides an interface between a local seth-repo and the Seth server. ceth helps users to manage:

  • Nodes
  • Cookbooks and recipes
  • Roles
  • Stores of JSON data (data bags), including encrypted data
  • Environments
  • Cloud resources, including provisioning
  • The installation of the seth-client on management workstations
  • Searching of indexed data on the Seth server

The following sections describe functionality common to all ceth subcommands:

  • Working with ceth
  • Common Options

ceth includes the following sub-commands:

Sub-command Description
ceth bootstrap The ceth bootstrap subcommand is used to run a bootstrap operation that installs the seth-client on the target system. The bootstrap operation must specify the IP address or FQDN of the target system.
ceth client The ceth client subcommand is used to manage an API client list and their associated RSA public key-pairs. This allows authentication requests to be made to the Seth server by any entity that uses the seth Server API, such as the seth-client and ceth.
ceth configure The ceth configure subcommand is used to create the ceth.rb and client.rb files so that they can be distributed to workstations and nodes.
ceth cookbook The ceth cookbook subcommand is used to interact with cookbooks that are located on the Seth server or the local seth-repo.
ceth cookbook site The ceth cookbook site subcommand is used to interact with cookbooks that are located at http://community.opscode.com/. A user account is required for any community actions that write data to this site. The following arguments do not require a user account: download, search, install, and list.
ceth data bag The ceth data bag subcommand is used to manage arbitrary stores of globally available JSON data.
ceth delete The ceth delete subcommand is used to delete an object from a Seth server. This subcommand works similar to ceth cookbook delete, ceth data bag delete, ceth environment delete, ceth node delete, and ceth role delete, but with a single verb (and a single action).
ceth deps The ceth deps subcommand is used to identify dependencies for a node, role, or cookbook.
ceth diff The ceth diff subcommand is used to compare the differences between files and directories on the Seth server and in the seth-repo. For example, to compare files on the seth server prior to an uploading or downloading files using the ceth download and ceth upload subcommands, or to ensure that certain files in multiple production environments are the same. This subcommand is similar to the git diff command that can be used to diff what is in the seth-repo with what is synced to a git repository.
ceth download The ceth download subcommand is used to download roles, cookbooks, environments, nodes, and data bags from the Seth server to the current working directory. It can be used to back up data on the seth server, inspect the state of one or more files, or to extract out-of-process changes users may have made to files on the seth server, such as if a user made a change that bypassed version source control. This subcommand is often used in conjunction with ceth diff, which can be used to see exactly what changes will be downloaded, and then ceth upload, which does the opposite of ceth download.
ceth edit The ceth edit subcommand is used to edit objects on the Seth server. This subcommand works similar to ceth cookbook edit, ceth data bag edit, ceth environment edit, ceth node edit, and ceth role edit, but with a single verb (and a single action).
ceth environment The ceth environment subcommand is used to manage environments within a single organization on the Seth server.
ceth exec The ceth exec subcommand uses the ceth configuration file to execute Ruby scripts in the context of a fully configured seth-client. This subcommand is most often used to run scripts that will only access Seth server one time (or otherwise very infrequently). Use this subcommand any time that an operation does not warrant full usage of the ceth subcommand library.
ceth index rebuild The ceth index rebuild subcommand is used to rebuild the search indexes for the open source Seth server. This operation is destructive and may take some time.
ceth list The ceth list subcommand is used to view a list of objects on the Seth server. This subcommand works similar to ceth cookbook list, ceth data bag list, ceth environment list, ceth node list, and ceth role list, but with a single verb (and a single action).
ceth node The ceth node subcommand is used to manage the nodes that exist on a Seth server.
ceth raw The ceth raw subcommand is used to send a REST request to a specified path using the Seth Server API.
ceth recipe list The ceth recipe list subcommand is used to view all of the recipes that are on a Seth server. A regular expression can be used to limit the results to recipes that match a specific pattern. The regular expression must be within quotes and not be surrounded by forward slashes (/).
ceth role The ceth role subcommand is used to manage the roles that are associated with one or more nodes on a Seth server.
ceth search The ceth search subcommand is used run a search query for information that is indexed on a Seth server.
ceth show The ceth show subcommand is used to view the details of one (or more) objects on the Seth server. This subcommand works similar to ceth cookbook show, ceth data bag show, ceth environment show, ceth node show, and ceth role show, but with a single verb (and a single action).
ceth ssh The ceth ssh subcommand is used to invoke SSH commands (in parallel) on a subset of nodes within an organization, based on the results of a search query.
ceth status The ceth status subcommand is used to display a brief summary of the nodes on a Seth server, including the time of the most recent successful seth-client run.
ceth tag The ceth tag subcommand is used to apply tags to nodes on a Seth server.
ceth upload The ceth upload subcommand is used to upload roles, cookbooks, environments, and data bags to the Seth server from the current working directory in the seth-repo. This subcommand is often used in conjunction with ceth diff, which can be used to see exactly what changes will be uploaded, and then ceth download, which does the opposite of ceth upload.
ceth user The ceth user subcommand is used to manage the list of users and their associated RSA public key-pairs.
ceth xargs The ceth xargs subcommand is used to build and execute command lines against objects on a Seth server using standard input.
